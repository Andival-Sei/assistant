## Типы документации

### Specs (Спецификации)

**Цель:** Описать требования и дизайн модулей  
**Формат:** User stories, модели данных, API  
**Примеры:** Спецификация модуля финансов, модель данных здоровья

### Guides (Руководства)

**Цель:** Помочь пользователю достичь конкретной цели  
**Формат:** Пошаговые инструкции, туториалы  
**Примеры:** «Как настроить проект», «Как добавить новый модуль»

### Concepts (Концепции)

**Цель:** Объяснить, как и почему что-то работает  
**Формат:** Объяснения, диаграммы, архитектурные решения  
**Примеры:** «Архитектура кеширования», «Как работает синхронизация»

### Reference (Справочник)

**Цель:** Описать факты о системе  
**Формат:** Структурированная информация, таблицы, схемы  
**Примеры:** «Конфигурация», «API endpoints»

### Processes (Процессы)

**Цель:** Описать процессы и стандарты команды  
**Формат:** Workflow, соглашения, best practices  
**Примеры:** «Git workflow», «Соглашение о коммитах»

### Meta (Мета)

**Цель:** Планирование и контекст  
**Формат:** Roadmap, ADR, информация для AI  
**Примеры:** «План развития», «Архитектурные решения»

## Вклад в документацию

При добавлении новой документации:

1. Определите тип документа (spec, guide, concept, reference, process, meta)
2. Используйте шаблон из `.templates/`
3. Разместите файл в соответствующей папке
4. Добавьте ссылку в этот README
5. Следуйте существующим шаблонам и формату
