# Architecture Decision Records (ADR)

Этот каталог содержит записи об архитектурных решениях проекта.

## Что такое ADR?

ADR (Architecture Decision Record) — документ, фиксирующий важное архитектурное решение:

- **Контекст** — почему возникла необходимость принять решение
- **Решение** — что было выбрано и почему
- **Последствия** — что это означает для проекта

## Зачем нужны ADR?

1. **Память проекта** — понять, почему было принято решение
2. **Онбординг** — новые разработчики понимают контекст
3. **Обсуждение** — структурированный формат для дискуссий
4. **Отслеживание** — видно, когда решения устаревают

## Статусы

| Статус         | Описание                  |
| -------------- | ------------------------- |
| **Proposed**   | Предложено, обсуждается   |
| **Accepted**   | Принято, реализуется      |
| **Deprecated** | Устарело, но ещё работает |
| **Superseded** | Заменено другим ADR       |

## Как добавить ADR

1. Скопируйте шаблон из [`../.templates/adr.md`](../../.templates/adr.md)
2. Назовите файл `XXXX-краткое-название.md` (следующий номер)
3. Заполните шаблон
4. Добавьте запись в индекс выше
5. Создайте PR для обсуждения (если применимо)

## Формат номера

- 4 цифры с ведущими нулями: `0001`, `0002`, etc.
- Позволяет до 9999 ADR (более чем достаточно)
