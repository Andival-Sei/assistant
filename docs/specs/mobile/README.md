# Mobile Specs (Android Native)

> Статус Android-клиента: нативное приложение сейчас **на паузе**. Эти спецификации остаются источником истины для backend-контрактов Web + Android, но реализация мобильного клиента не в приоритете.

Документы фиксируют единые backend-контракты для Web + Android.

## Состав

- `auth.md` — сессии, логин/регистрация, хранение токена.
- `finance.md` — кошельки/категории/транзакции/split операции.
- `health.md` — метрики, интеграции, sync, retry/backoff и Health Connect.

## Политика конфликтов

- Источник истины: Supabase.
- При конфликте приоритет у записи с более новым `updated_at`.
- Все upsert в health должны быть идемпотентными по `user_id + recorded_for + source`.

## Retry/backoff

- Edge Functions sync: экспоненциальный backoff, старт 30с, максимум 30мин.
- При `429` учитывать `retry_after_seconds`.
- Повтор синка не должен дублировать записи.
